# สกิล: ตรวจสอบความครบถ้วนของข้อมูลภาษาไทย

ใช้เมื่อดึงข้อมูลจาก Chat, Cowork, Project Knowledge, PDF, Word, OCR, Google Docs หรือไฟล์แนบทุกประเภท

---

## ลำดับความสำคัญ (Priority Order)

1. **ความครบถ้วนของข้อมูล** (Completeness) — มาก่อนเสมอ
2. **ความถูกต้องของภาษาไทย** (Accuracy)
3. **การรักษารูปแบบเอกสาร** (Formatting)
4. **ความสวยงามในการจัดวาง** (Presentation)
5. **ความเร็วในการประมวลผล** (Speed)

> ห้ามแลกความครบถ้วนของข้อมูลกับความเร็วในการตอบ

---

## กฎความครบถ้วนของข้อมูล

ห้ามละเว้นสิ่งต่อไปนี้:

```
□ ข้อความใดๆ
□ ย่อหน้า
□ หัวข้อหลัก / หัวข้อย่อย
□ รายการ Bullet
□ ตาราง
□ ข้อความท้ายหน้า / ต้นหน้าใหม่
□ ข้อความในกล่องข้อความ (Text Box)
□ หมายเหตุ / Footnote (ถ้าอ่านได้)
□ ข้อความจาก Chat / Cowork ทุกส่วน
```

---

## Checklist ก่อนส่งผลลัพธ์

ตรวจสอบทุกข้อก่อนตอบ:

```
✓ จำนวนหัวข้อหลักครบถ้วน
✓ จำนวนหัวข้อย่อยครบถ้วน
✓ รายการ Bullet ไม่หาย
✓ ลำดับเลขข้อไม่ขาด
✓ ตารางไม่ตกหล่น
✓ ข้อความจาก Chat/Cowork ถูกนำมาประมวลผลครบทุกส่วน
✓ ไม่มีการตัดข้อความเพราะข้อจำกัดด้านการจัดรูปแบบ
```

---

## กรณีข้อมูลยาวเกินขีดจำกัด

**ห้ามตัดเนื้อหาออกเอง** — ให้แบ่งผลลัพธ์เป็นส่วนย่อย:

```
ตอนที่ 1/3 → ส่ง
ตอนที่ 2/3 → ส่ง
ตอนที่ 3/3 → ส่ง
```

- ห้ามสรุปแทนเนื้อหาต้นฉบับ
- ห้ามละเว้นส่วนใดโดยไม่แจ้ง

---

## การตรวจจับข้อความตกหล่น

หากพบสัญญาณดังต่อไปนี้ให้แจ้งทันที:

| สัญญาณ | ตัวอย่าง |
|--------|---------|
| ประโยคขาดกลางคัน | "ราคาเริ่มต้นที่..." (จบผิดปกติ) |
| หัวข้อกระโดดข้ามลำดับ | หัวข้อ 1 → หัวข้อ 3 (ขาดหัวข้อ 2) |
| เลขข้อหาย | ข้อ 1, 2, 4, 5 (ขาดข้อ 3) |
| ตารางไม่สมบูรณ์ | คอลัมน์หรือแถวขาดหาย |
| ย่อหน้าถูกตัด | เนื้อหาจบกะทันหันโดยไม่มีบทสรุป |
| ข้อความจบผิดปกติ | ประโยคไม่สมบูรณ์ |

**ข้อความแจ้งเตือนที่ต้องใช้:**

```
[พบความเป็นไปได้ว่าข้อมูลต้นฉบับตกหล่น โปรดตรวจสอบไฟล์ต้นฉบับ]
```

แจ้งก่อนดำเนินการต่อเสมอ
