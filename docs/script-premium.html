SCENE 1 แนะนำตัวและภาพรวมระบบ
⏱ 0:00 – 0:30
🎯 ทำให้ผู้ดูเข้าใจว่าวิดีโอนี้สอนอะไร และระบบ Premium ทำงานอย่างไรโดยรวม
สวัสดีครับ วันนี้เราจะมาดูการใช้งานเมนู QR Code แพ็ก Premium ด้วยกันนะครับ

ระบบ Premium นี้ครับ จะช่วยให้ลูกค้าของเราสามารถสแกน QR Code ที่โต๊ะ แล้วก็เลือกเมนูได้เลยบนมือถือ โดยที่ออเดอร์จะเข้า LINE OA ของร้านอัตโนมัติ ไม่ต้องรับโทรศัพท์ ไม่ต้องจดออเดอร์เองครับ

นอกจากนี้ ระบบ Premium ยังรองรับ Delivery, 2 ภาษา ไทย/English, ตัวเลือกเมนู และแก้ไขเมนูเองได้ผ่าน Supabase โดยไม่ต้องรอช่างด้วยครับ

ในวิดีโอนี้เราจะใช้เวลาประมาณ8 นาทีครับ ไปดูกันเลย
💡 Tips ผู้สอน
เปิดหน้าเมนูไว้บนหน้าจอก่อนเริ่มอัด
พูดช้าๆ ชัดๆ ช่วง Intro นี้สำคัญมาก
แนะนำตัวสั้นๆ ก็ได้ เช่น "ผม…จาก khaomoo-menu"
SCENE 2 สาธิต Dine-in — สแกน QR → สั่งอาหาร → LINE OA
⏱ 0:30 – 2:00
🎯 แสดงขั้นตอนการสั่งอาหารแบบนั่งในร้านตั้งแต่ต้นจนจบ
เริ่มจากการสั่งอาหารแบบ Dine-in นะครับ สมมติว่าลูกค้านั่งอยู่ที่โต๊ะ 1 ครับ

⚡ Action: หยิบมือถือ / เปิดกล้องสแกน QR Code โต๊ะ 1

ลูกค้าจะสแกน QR Code ที่วางบนโต๊ะครับ พอสแกนแล้วก็จะเข้ามาหน้าเมนูทันที โดยระบบจะรู้ว่าเป็นโต๊ะ 1 อัตโนมัติครับ ลูกค้าไม่ต้องกรอกอะไรเพิ่มเลย

⚡ Action: แสดงหน้า Category — กดเลือกหมวดเมนู

ในหน้าเมนูครับ ลูกค้าเลือกหมวดหมู่ได้ แล้วก็กดรายการที่ต้องการ ถ้ามีตัวเลือก เช่น ขนาด หรือ เผ็ดน้อย/มาก ก็จะขึ้นให้เลือกตรงนี้เลยครับ

⚡ Action: เลือก Options → กด "เพิ่มลงตะกร้า"

เมื่อเลือกครบแล้ว กด"ยืนยันออเดอร์"ครับ ระบบจะส่งออเดอร์ไปที่ LINE OA ของร้านทันทีเลย

⚡ Action: เปิด LINE OA บนมือถือ/คอม แสดงข้อความออเดอร์

ดูครับ — ออเดอร์เข้ามาใน LINE OA ร้านเรียบร้อยแล้ว มีทั้งหมายเลขโต๊ะ รายการอาหาร และตัวเลือกครบถ้วนเลยครับ ทีมร้านก็สามารถเริ่มทำอาหารได้เลย
💡 Tips ผู้สอน
ซ้อมกดเมนูก่อนอัดจริง ไม่ให้ติดขัด
แสดง Options อย่างน้อย 1 ตัวเลือก เช่น ขนาดเล็ก/ใหญ่
zoom เข้าที่ LINE เพื่อให้เห็นข้อความชัดๆ
เน้นว่า "ออเดอร์เข้าอัตโนมัติ ไม่ต้องทำอะไรเพิ่ม"
SCENE 3 สาธิต Delivery — กรอกที่อยู่ → ออเดอร์เข้า LINE OA
⏱ 2:00 – 3:30
🎯 แสดงว่าระบบ Delivery ทำงานต่างจาก Dine-in อย่างไร
ต่อมาดูฝั่งDelivery กันบ้างนะครับ

⚡ Action: เปิดลิงก์เมนู Delivery หรือกดปุ่ม "Delivery" ในแอป

ลูกค้าที่สั่งแบบเดลิเวอรีจะเข้าผ่านลิงก์เมนูที่ร้านแชร์ไว้ครับ พอเข้ามาแล้ว ก็เลือกเมนูได้เหมือนกันเลย

⚡ Action: เลือกเมนู → กด "ดำเนินการสั่ง"

พอเลือกเมนูครบแล้ว ระบบจะให้กรอกชื่อ เบอร์โทร และที่อยู่สำหรับจัดส่งครับ

⚡ Action: กรอกฟอร์ม Delivery → กด "ยืนยัน"

กด"ยืนยันออเดอร์"ครับ ออเดอร์ก็จะเข้า LINE OA เหมือนกัน แต่ครั้งนี้จะมีชื่อ เบอร์ และที่อยู่ของลูกค้ามาด้วยเลยครับ

⚡ Action: เปิด LINE OA ดูข้อความ Delivery

ดูนะครับ — ออเดอร์ Delivery เข้ามาแล้ว ทางร้านก็รับรู้ว่าต้องส่งไปที่ไหน ง่ายมาก ไม่ต้องโทรหากันเลยครับ
💡 Tips ผู้สอน
กรอกที่อยู่ปลอมไว้ล่วงหน้า ไม่ต้องพิมพ์จริงระหว่างอัด
เปรียบเทียบ LINE message Dine-in vs Delivery ให้เห็นความต่าง
บอกว่าร้านสามารถ copy ที่อยู่ไปใส่ใน Google Maps ได้เลย
SCENE 4 จัดการเมนูใน Supabase — แก้ราคา / ซ่อนเมนู
⏱ 3:30 – 5:00
🎯 แสดงว่าร้านสามารถอัปเดตเมนูเองได้โดยไม่ต้องรอช่าง
ฟีเจอร์ที่หลายคนชอบมากครับ — แก้ราคาหรือซ่อนเมนูเองได้ผ่าน Supabase โดยไม่ต้องรอช่างหรือจ่ายเพิ่มครับ

⚡ Action: เปิด supabase.com → เข้า project ของร้าน

เข้าไปที่ supabase.com ครับ Login ด้วย email ที่เราส่งให้ แล้วก็เข้า project ของร้านครับ

⚡ Action: ไปที่ Table Editor → เลือก table "menu_items"

ในหน้า Dashboard ไปที่ Table Editor ครับ แล้วเลือก table ชื่อ menu_items ตรงนี้จะเห็นรายการเมนูทั้งหมดของร้านครับ

สมมติว่าเราอยากแก้ราคา "ข้าวหมูแดง" จาก 60 เป็น 65 บาทครับ
⚡ Action: ดับเบิลคลิกช่อง price ของเมนู → แก้ตัวเลข → กด Save

ดับเบิลคลิกที่ช่อง price แก้ตัวเลข กด Save — เท่านี้เลยครับ ลองเปิดหน้าเมนูใหม่ดู ราคาอัปเดตทันทีเลย

ถ้าอยากซ่อนเมนูชั่วคราว เช่น หมดแล้ว ก็แค่เปลี่ยนช่อง is_available จาก true เป็น false ครับ เมนูนั้นก็จะหายไปจากหน้าแอปทันทีเลย
💡 Tips ผู้สอน
เตรียม tab Supabase + tab เมนูแอปไว้คู่กัน เพื่อแสดง real-time update
แสดง is_available = false จริงให้เห็นว่าเมนูหายจากแอป
เน้น: "ไม่ต้องมีความรู้ coding เลย แค่ดับเบิลคลิกแก้ตัวเลข"
อย่าลืม reset ราคาหลังจากสาธิต (ถ้าเป็น live data)
SCENE 5 สลับภาษา ไทย / EN — Bilingual Feature
⏱ 5:00 – 6:00
🎯 แสดงว่าการสลับภาษาเปลี่ยนอะไรบ้างในแอป
แพ็ก Premium รองรับ2 ภาษา ไทยและอังกฤษครับ เหมาะมากสำหรับร้านที่มีลูกค้าต่างชาติมาใช้บริการ

⚡ Action: เปิดหน้าเมนู → กดปุ่ม EN ที่มุมขวาบน

ในแอปเมนูจะมีปุ่มสลับภาษาอยู่มุมขวาบนครับ กด EN ปุ๊บ — ชื่อเมนู คำอธิบาย หมวดหมู่ และป้ายต่างๆ จะเปลี่ยนเป็นภาษาอังกฤษทันทีเลย

เช่น "ข้าวหมูแดง" จะเปลี่ยนเป็น "Red Pork Rice" ครับ หมวด "อาหารจานเดียว" จะเปลี่ยนเป็น "Main Course" เป็นต้น

⚡ Action: กด TH เพื่อสลับกลับ → แสดงทั้งสองภาษาสลับกัน

กดสลับกลับได้เลยครับ ลูกค้าเลือกภาษาที่ตัวเองถนัดได้ตามใจ ข้อมูลทั้งหมดนี้เราตั้งค่าไว้ตอนทำเมนูให้แล้วครับ ไม่ต้องทำอะไรเพิ่ม
💡 Tips ผู้สอน
แสดงการสลับ 2-3 ครั้ง ให้เห็นว่าเร็วและ smooth
เปรียบเทียบ ชื่อภาษาไทย vs EN ควบคู่กัน
ถ้าร้านไม่มีลูกค้าต่างชาติ บอกว่าฟีเจอร์นี้ไม่บังคับใช้
SCENE 6 QR Code แยกโต๊ะ + Zoom Training
⏱ 6:00 – 7:00
🎯 แสดง QR แต่ละโต๊ะที่ไฟล์ส่งมอบ และแนะนำ Zoom training
ในแพ็ก Premium เราจะให้QR Code แยกต่างหากทุกโต๊ะ รองรับสูงสุด 30 โต๊ะครับ

⚡ Action: เปิดโฟลเดอร์ QR Code PNG ที่ส่งให้ลูกค้า

เราจะส่งไฟล์ PNG ของ QR Code ครบทุกโต๊ะให้ครับ ร้านก็นำไปปริ้นท์ใส่กรอบหรือแลมิเนตวางบนโต๊ะได้เลย แต่ละ QR จะ link ไปยัง URL ที่ระบุหมายเลขโต๊ะ เช่น /table/1, /table/2 ครับ

และหลังจากเราส่งมอบงานแล้ว เราจะนัดZoom สอนการใช้งาน 30 นาทีครับ จะสอนดูออเดอร์ใน LINE, แก้เมนูใน Supabase, และตอบทุกคำถามที่สงสัยได้เลย

ถ้าพร้อมแล้วก็แจ้งมาทางLINE @khaomoo เพื่อนัดวันได้เลยครับ
💡 Tips ผู้สอน
แสดง QR หลายโต๊ะในโฟลเดอร์ให้เห็นว่าส่งมาครบ
แนะนำให้ปริ้นท์ A5 หรือ A6 แล้วแลมิเนต ทนทาน
เน้นว่า Zoom ฟรี รวมอยู่ในแพ็กแล้ว
